# 问卷系统项目 README


## 项目概述
本项目是集**问卷创建管理、填写、用户权限管控、数据可视化**于一体的系统，由3名成员分工开发。


## 项目成员分工
| 成员 | 负责模块                     |
|------|------------------------------|
| A    | 问卷创建 & 管理模块负责人     |
| B    | 问卷填写 & 并发网络访问负责人 |
| C    | 用户系统 & 权限 & 可视化负责人 |


## 各模块任务清单
### A（问卷创建&管理模块）
#### 程序开发任务
| 任务项               | 内容                                                                 |
|----------------------|----------------------------------------------------------------------|
| 问卷创建（核心）| 实现问卷创建/发布/关删界面；支持多题型；设计问卷数据结构；数据库CRUD |
| 问卷编辑功能         | 支持问卷修改/复制/模板复用                                           |
| 违规问卷检测         | 实现违规判断；提供管理员删问卷接口（与B协作）|
| 网络模块             | 开发问卷创建/提交接口；保证数据一致性（与B协同）|

#### 数据库任务
| 时间 | 任务名               | 内容                                                                 |
|------|----------------------|----------------------------------------------------------------------|
| 第1周 | 问卷核心表设计       | 设计`survey/question/option`表；定义主外键、字段约束                 |
| 第2周 | 问卷增删改查实现     | 封装Python函数/类；提供API给B调用                                    |
| 第3周 | 违规问卷标记表设计   | 设计`violation`表；关联`survey`表；实现标记CRUD                      |
| 第4周 | 数据库调试与优化     | 测试接口；调整结构/索引；处理多线程冲突（与B协作）|

#### 报告撰写责任
| 报告章节       | 内容                                                                 |
|----------------|----------------------------------------------------------------------|
| 第3章（项目内容） | 问卷创建模块设计+功能；问卷数据库结构（含ER图）；违规检测逻辑         |
| 第4章（项目结果） | 展示问卷创建界面/流程；数据库结构/流程图                             |
| 第5章（分工说明） | 描述负责模块的难点与实现                                             |


### B（问卷填写&并发访问模块）
#### 程序开发任务
| 任务项               | 内容                                                                 |
|----------------------|----------------------------------------------------------------------|
| 问卷填写（核心）| 实现填写界面；输入校验；答案写入数据库；填写记录CRUD                 |
| 多线程并发访问       | 搭建网络服务；多线程处理填写请求；实现锁机制避免冲突                 |
| 数据清洗模块         | 处理空值/去重/非法内容；应对重复/异常提交                           |
| 全局数据查看         | 提供管理员查看填写记录接口；支持条件筛选（与A、C协作）|

#### 数据库任务
| 时间 | 任务名               | 内容                                                                 |
|------|----------------------|----------------------------------------------------------------------|
| 第1周 | 答案表设计           | 设计`answer`表；关联`question_id/user_id`；确定字段约束              |
| 第2周 | 填写数据增删改查     | 实现答案写入；提供查询函数；增加数据校验                             |
| 第3周 | 并发控制             | 多线程写数据库；用事务/锁保证一致性；测试并发场景                   |
| 第4周 | 数据清洗&测试        | 清洗`answer`表数据；与可视化模块联调统计数据                           |

#### 报告撰写责任
| 报告章节       | 内容                                                                 |
|----------------|----------------------------------------------------------------------|
| 第3章（项目内容） | 问卷填写模块流程；网络开发设计；数据一致性与锁机制                   |
| 第4章（项目结果） | 并发测试截图；填写界面截图                                           |
| 第5章（分工说明） | 描述负责模块的实现过程与难点                                         |

# A-distributed-questionnaire-management-system-based-on-Python
PySurvey, a C/S-based questionnaire management system, supports multi-user simultaneous online access. It enables socket connections, questionnaire creation/filling, record query, data visualization, and digitalizes full-life-cycle questionnaire management.
